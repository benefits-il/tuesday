# מפה לבן בלבד — ערמת ה-onboarding של TUESDAY

זו לא ערמה אקראית. זו ערמה מבוימת. רוב הקבצים אמיתיים ושימושיים, וכמה מהם רעש מכוון שנפל לאותה תיקייה. המטרה הפדגוגית היא בדיוק זה, שהלומד יראה שסוכן Point צריך לסנן ולסדר, לא רק לבלוע הכל.

הדמו: התרחיש הוא עובד חדש שמתחיל ב-TUESDAY ביום שלישי. HR או המייסד מרכיבים מתוך הערמה מצגת onboarding כיפית, ש-POWER יהפוך אחר כך לעמוד HTML מונפש.

## אות (signal) — שייך למצגת

| קובץ | מה זה | מה Point אמור למשוך מזה |
|---|---|---|
| hr-welcome-note.md | פתק קליטה חם מ-HR (דנה) | הטון האנושי-יבש לפתיחה, המסר ששבוע ראשון לא מבחן, מותר לשאול |
| founder-note.md | פתק מהמייסד (יואב), כתוב בקול ה-welcome-email | הסיפור והאתוס, "אין לך מנהל אתה המנהל", שקף ברוך הבא חזק |
| culture-and-values.md | מסמך ערכים, איך הצוות עצמו עובד | עמוד הערכים, חמשת העקרונות: אחריות, דיבור ישר, אנטי-בירוקרטיה, תוצאות, הומור עצמי |
| first-week-checklist.md | צ'קליסט שבוע ראשון לפי ימים | שקף "מה קורה השבוע", מבנה יום-ראשון / אמצע / סוף |
| tools-and-access.md | מה העובד מקבל ואיפה הכל גר | שקף הגישות והכלים, איפה מתנהלים דברים |
| team-intro-blurbs.md | שש שורות היכרות עם הצוות | שקף "מי האנשים", פרצופים ושמות |
| perks-and-how-we-work.md | תנאים, גמישות, עבודה מהבית, הטבות | שקף "איך מתנהלים פה", שעות, חופשות, הדברים הקטנים |
| tuesday-brand-quickref.md | תזכורת מותג מקוצרת | לא תוכן למצגת אלא חוקי המראה: צהוב/שחור/לבן, רונה Tiwaz, טיפוגרפיה כבדה, קול יבש |

## רעש (noise) — לא שייך, אמור להיזרק

| קובץ | מה זה | למה זה רעש |
|---|---|---|
| snacks-poll-thread.md | שרשור סלאק על מכונת קפה וחטיפים | התלבטות משרדית חולפת, אפס קשר לקליטת עובד |
| it-password-rotation-memo.md | מזכר IT רבעוני על החלפת סיסמאות (באנגלית) | תהליך תחזוקה תקופתי, נשלח לכולם, לא חלק מ-onboarding |

## איך נראה פלט Point טוב

- מזהה ומתעלם משני קבצי הרעש, ולא תוחב מכונת קפה או החלפת סיסמאות למצגת ברוך הבא.
- מראיין את בן לפני שהוא בונה, לא קופץ ישר לתוצר.
- מוציא handoff אחד בלבד בפורמט MD, מבנה שקף-אחר-שקף, בלי HTML ובלי עיצוב בשלב הזה.
- שומר על הקול היבש-ישיר ועל פלטת שלושת הצבעים מתזכורת המותג.
- קשת שקפים סבירה: ברוך הבא ← מי אנחנו והערכים ← מי האנשים ← מה קורה השבוע ← כלים וגישות ← איך מתנהלים ← סיום מעודד.

## הצעד הבא בקורס

הפרומפטים המוכנים נמצאים ב-_prompts.md. שלב א ב-Point מייצר את המבנה, שלב ב ב-POWER הופך אותו לעמוד HTML מונפש. זה הרגע שבו הלומדים פוגשים HTML בפעם הראשונה.
